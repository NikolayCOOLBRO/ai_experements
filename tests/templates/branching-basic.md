# Branching: Basic Checkpoint Flow

## Настройки агента

- Режим контекста: Ветки диалога

## Шаги

1. Создать агента и новый чат.
2. Отправить сообщение: `Запомни, что проект называется Nova Desk, а приоритет релиза - 12 июля. Ответь: записал.`
3. Отправить сообщение: `Запомни, что целевая аудитория - B2B-команды продаж. Ответь: записал.`
4. Нажать `Сохранить ветку отсюда`.
5. Создать первую ветку из checkpoint с названием `Ветка A`.
6. Создать вторую ветку из того же checkpoint с названием `Ветка B`.

## Ожидание

- Один checkpoint доступен в списке контрольных точек.
- Из одного checkpoint можно создать минимум две разные ветки.
- Обе ветки появляются в списке чатов.
- В обеих ветках видна одинаковая начальная история до checkpoint.
- В обеих ветках пока нет новых сообщений после точки ветвления.
